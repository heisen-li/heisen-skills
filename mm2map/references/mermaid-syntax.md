## Mermaid Mindmap 语法参考

### 基本结构

```mermaid
mindmap
  root((根节点))
    子节点1
      孙节点1a
      孙节点1b
    子节点2
      孙节点2a
    子节点3
```

### 语法规则

1. **声明类型**: 必须以 `mindmap` 开头
2. **根节点**: 用 `root((...))` 双圆括号包裹，表示圆形根节点
3. **层级缩进**: 用空格缩进表示父子关系，每层缩进2个空格
4. **节点文本**: 直接写在缩进后，无需引号或括号
5. **节点形状**:
   - `((文本))` — 圆形（仅用于根节点）
   - `(文本)` — 圆角矩形
   - `[文本]` — 矩形
   - 默认无括号 — 矩形

### 示例：中文思维导图（详尽版）

**重要：节点文本应包含完整定义和关键信息，而非简短关键词。**

```mermaid
mindmap
  root((人工智能：使计算机模拟人类智能行为的科学与技术))
    机器学习：从数据中学习模式并做出预测的算法体系
      深度学习：基于多层神经网络的机器学习方法
        CNN卷积神经网络：擅长图像特征提取，用于图像识别和目标检测
        RNN循环神经网络：擅长序列数据处理，用于语音识别和文本生成
        Transformer：基于自注意力机制，GPT/BERT等大模型的基础架构
      监督学习：有标签数据训练，输入输出映射关系明确
        分类任务：预测离散类别标签
        回归任务：预测连续数值
      无监督学习：无标签数据训练，发现数据内在结构
        聚类：将相似数据归为同一组
        降维：压缩数据维度保留关键信息
    自然语言处理NLP：让计算机理解和生成人类语言
      文本分类：将文本自动归类到预定义类别
      机器翻译：自动将一种语言翻译为另一种语言
      问答系统：根据用户提问从知识库中检索并生成答案
    计算机视觉CV：让计算机理解和处理图像/视频
      图像识别：识别图像中的物体类别
      目标检测：定位图像中物体的位置和类别
      图像分割：将图像像素划分为不同语义区域
    应用场景
      医疗诊断：AI辅助影像诊断，提高诊断效率和准确率
      自动驾驶：感知-决策-控制的完整技术链
      智能客服：基于NLP的自动对话回复系统
```

### 禁止事项

- 节点文本中不要使用 `{ } [ ] # |` 这些特殊字符（Mermaid 会误解析）
- 不要在节点文本中使用换行符
- 不要在同一节点中使用括号嵌套，如 `((外((内)))` 不合法
- **深度无上限**：完整保留源内容的层级结构，不做截断
- **广度无上限**：每个父节点下的子节点数量不限，忠实呈现所有知识点

### 特殊字符处理

如果节点文本必须包含特殊字符，用以下替换：

| 原字符 | 替换 |
|--------|------|
| `{` `}` | 用中文 `{` `}` 或删除 |
| `[` `]` | 用中文括号或删除 |
| `#` | 用 `No.` 或中文 `第` |
| `|` | 用 `-` 或 `或` |

### 渲染方式

1. **mermaid.live** — 在线渲染器: https://mermaid.live
2. **Typora** — Markdown编辑器，直接粘贴渲染
3. **Obsidian** — 安装 Mermaid 插件后渲染
4. **VS Code** — 安装 "Mermaid Markdown Syntax Highlighting" 扩展
5. **HTML** — 使用 mermaid.js CDN:
   ```html
   <script src="https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js"></script>
   <div class="mermaid">
   mindmap
     root((主题))
       ...
   </div>
   ```
